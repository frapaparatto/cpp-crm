**What I built:**

- `policy.hpp`: Policy class with PolicyStatus enum, client_uuid as foreign reference, constructor
- `interaction.hpp`: Interaction abstract base, Appointment and Contract derived classes with display() const override

**What I learned:**
- Rule of Zero: no custom destructor needed when no manual resources exist, std::string handles its own memory
- `const&` return for strings avoids copies while preventing modification, primitives return by value because const on a copy is meaningless
- Pure virtual methods make a class abstract and uninstantiable, only derived classes can be constructed
- Virtual destructor is required on any base class, without it deleting through a base pointer causes UB
- `override` lets the compiler verify the signature actually matches the base, catches typos silently missed otherwise
- Protected constructor on base class, callable only from derived constructors, never from outside
- email-to-uuid resolution belongs in the service layer, not in domain object constructors

**What I built:**

- `i_policy_repository.hpp`: abstract interface with insert, remove, update, findByClientUuid, findAll
- `i_interaction_repository.hpp`: abstract interface with insert, remove, update, filter by type and date, findByClientUuid, findAll

**What I learned:**
- `std::optional<T&>` is forbidden: optional means "maybe nothing", reference means "always something", the two contradict each other
- `std::optional` usage: has_value(), value(), value_or(), if (result) as shorthand check
- Abstract types cannot be stored in vectors by value, use `std::vector<std::unique_ptr<Base>>` instead
- Object slicing: copying a derived object into a base slot loses the derived fields silently, with abstract types the compiler refuses the copy entirely
- To read polymorphic objects pass `const Base&`, to own and store use `std::unique_ptr<Base>`
- Repository pattern: the repository is the source of truth and retains ownership, callers receive copies or snapshots
- `findAll` and filter methods return empty vector not `std::optional<vector>`, an empty result is not a failure
