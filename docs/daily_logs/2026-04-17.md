**What I learned:**

Move semantics and value categories:
- `std::move` is a cast, not an operation. It casts a named lvalue to an rvalue reference, telling the compiler the object can be stolen from. The actual stealing happens in the move constructor or move assignment of the receiving type.
- `const` and move are incompatible. Move requires modifying the source. A const object cannot be modified, so move silently falls back to copy.
- Copy constructor is called during initialization when the destination does not exist yet. Copy assignment is called when the destination already exists and existing resources must be freed before accepting new ones.
- Assignment cannot be elided. Elision only works during initialization when the compiler controls where the object is built. During assignment the destination already exists at a fixed memory address.
- `noexcept` is a runtime contract, not a compile-time check. If a `noexcept` function throws, `std::terminate` is called immediately. The program ends rather than propagating corrupted state.
- `std::vector` reallocation checks whether the move constructor is `noexcept`. If yes, it moves elements into the new buffer by stealing pointers. If no, it falls back to copying because a throwing move halfway through reallocation leaves the vector in an unrecoverable state.
- `Client` follows rule of zero and `std::string` has a `noexcept` move constructor, so the compiler-generated move constructor for `Client` is automatically `noexcept`.
- `emplace_back` constructs directly inside vector memory, eliminating even the move. For `Client` with two constructors and many parameters, `push_back` with `std::move` is the practical choice.

Small String Optimization:
- SSO stores short strings directly inside the string object using a union of two storage strategies sharing the same memory region. A flag determines which interpretation is active.
- On a 64-bit system a pointer is 8 bytes and `size_t` is 8 bytes. The allocated storage struct occupies 16 bytes. Short strings reuse those same 16 bytes as a char array, storing up to 15 characters plus a null terminator with no heap allocation.
- Moving a short string is not cheaper than copying it because there is no heap buffer to steal. The data lives directly in the object. Move semantics provide real benefit only for strings that exceed the SSO threshold.
- For `Client` fields like names, city, and postal code, most values fit within SSO. Notes and addresses may exceed it.
